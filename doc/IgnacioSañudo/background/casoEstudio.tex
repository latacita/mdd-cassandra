Esta sección presenta \emph{Twissandra}, el caso de estudio que se utilizará lo largo de este proyecto. Twissandra es un proyecto creado para aprender como utilizar Cassandra. El modelo UML correspondiente a Twissandra se puede ver en la figura~\ref{back:fig:twissandra}. Twissandra es una versión simplificada de Twitter.

Twitter\footnote{} es una red social de microblogging, actualmente está muy extendida, que permite escribir a sus usuarios pequeños mensajes de texto, denominados \emph{tweets}. Un \emph{tweet} es simplemente un texto con un límite de 140 caracteres publicado a una hora determinada. La colección de todos los tweets publicados por un usuario cronológicamente ordenados determinan su \emph{userline}.

Cada usuario registrado en Twitter puede seguir a otros usuarios registrados. Cuando decimos, por ejemplo, que Pedro sigue a María significa que Pedro recibirá en su cuenta todos los mensajes que publique María. En este caso, se dice que Pedro es un \emph{follower} de María. De esta forma, cada usuario tiene asociado un \emph{timeline} que no es más que la colección de todos los \emph{tweets} publicados por las personas a las que sigue ordenados cronológicamente, es decir, por fecha de publicación. 

Los principales casos de uso de \emph{Twissandra} son: 

\begin{enumerate}
    \item Obtener el \emph{timeline} de un usuario determinado. 
    \item Obtener el \emph{userline} de un usuario determinado. 
    \item Obtener la lista de usuarios que un usuario está siguiendo.  
    \item Obtener la lista de usuarios que están siguiendo a un usuario específico. 
\end{enumerate}

Las dos últimas listas deben se deben devolver ordenadas cronológicamente.
